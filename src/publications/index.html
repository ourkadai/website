Gain Without Pain on Spring Boot Microservices Journey at Java Z-Day

With all the recent hypes on Microservices, all Java Developers/Architects will now take the journey to microservices ( even if we don't need it :).) Microservices are supposed to make deployment easier which it may or may not as it varies teams to teams. But what is certain is that it makes development harder with distributed teams, complex deployment pipelines, code reusability, and on top of all service MESS. In this talk, we will discuss the common side effects of Microservice and how to gain without pain on this adventurous journey.
